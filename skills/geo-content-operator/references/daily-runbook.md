# Daily Runbook

Use this as the operator sequence before handing off to `geo-article-writer`.

## 1. Pull Signals

Collect available signals from the task context:

- keyword or topic
- business context
- audience and buyer role
- user-supplied evidence, URLs, examples, screenshots, or audit notes
- requested output type
- requested distribution platforms
- requested visuals

Optional when tools are available:

- search current SERP or AI answer behavior
- inspect competitor pages
- inspect adjacent-tool or ecosystem pages
- record cited sources and missing citation opportunities

Do not claim live reconnaissance happened unless a tool was used.

## 2. Check Portfolio State

Before approving the angle, inspect:

- `keyword-portfolio.md`
- `portfolio-memory.md`
- `citation-log.md`
- `promotion-briefs.md`

If project-specific state exists outside the bundled skill, prefer it over these seed files.

## 3. Sort The Signal

Classify the opportunity:

- reinforcement: deepen authority around an owned or target keyword
- expansion: cover an adjacent buyer question
- promotion: support a product, feature, or comparison page
- acquisition: capture a new problem-aware audience
- repair: fix an answer-engine citation or framing gap

## 4. Choose The Angle

The chosen angle must specify:

- reader problem
- core claim
- content lane
- mention intensity
- proof posture
- example policy
- visual policy

Avoid broad explainers unless the keyword truly needs a definition page.

## 5. Choose Asset Stack

Pick only useful assets:

- article
- comparison table
- diagnostic checklist
- before/after example
- framework diagram
- distribution hooks
- visual prompt and generated image

If an image is requested, choose 1-3 visuals only when each has a specific editorial job.

## 6. Package For Writer

The handoff should be short and operational. Include:

- the exact claim the article must defend
- why this article is not a repeat
- the examples it should use
- the facts it must not invent
- where metadata should be stored
- what checks decide pass/fail

## 7. Post-Run Update

For a real production workflow, update:

- portfolio memory with final title, thesis, lane, and hook
- keyword portfolio with status and asset role
- citation log if research or AI-engine checks were performed
- promotion briefs if the output supports a campaign
